2-celery生成环境使用

发表于 2019-12-21 | 分类于 python/celery

celery 生产环境使用

  • celery的配置文件统一放到settings目录中

  • 使用redis 作为broker 和 backed

  • 使用superisor统一维护worker,-c 指定了任务数,-n 制定了workname,-A指定celery APP
  • 在PYTHONPATH 所在路径范围编写异步任务处理函数 这将会与celert worker -A 对应,初始化好celery app后,直接使用修饰器即可使用celery
  • 使用rabbitmq作为broker,使用redis作为backed,通过routing_key设定不同异步函数的优先级,提升负载。
  • 使用superisor统一维护worker,-c 指定了任务数,-n 制定了workname,-A指定celery APP,-Q选项,指定了不同类型的worker绑定的任务,需要和celeryconfig.py对照

other

  • 监控,使用celery flower 监控任务执行情况。

  • 可以使用celery -A WORKER amqp queue.purge QUEUE_NAME 清空队列数据
